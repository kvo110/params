PHP Processing Template

Use this PHP file as your processing page. It shows the raw POST array, safely prints individual fields, and loops through all submitted values. 

<?php
function safe($value) {
    return htmlspecialchars($value ?? '', ENT_QUOTES, 'UTF-8');
}

if ($_SERVER["REQUEST_METHOD"] !== "POST") {
    echo "<h1>No form data received.</h1>";
    exit;
}

$name      = safe($_POST["name"] ?? "");
$email     = safe($_POST["email"] ?? "");
$address   = safe($_POST["address"] ?? "");
$ccard     = safe($_POST["ccard"] ?? "");
$dob       = safe($_POST["dob"] ?? "");
$gender    = safe($_POST["gender"] ?? "");
$phone     = safe($_POST["phone"] ?? "");
$track     = safe($_POST["track"] ?? "");
$statement = safe($_POST["statement"] ?? "");
?>

<body>
  <div class="container">
    <h1>Processed Form Submission</h1>

    <h2>Raw Form Data</h2>
    <pre><?php print_r($_POST); ?></pre>

    <h2>User Signup Details</h2>
    <table>
      <tr><th>Name</th><td><?php echo $name; ?></td></tr>
      <tr><th>Email</th><td><?php echo $email; ?></td></tr>
      <tr><th>Address</th><td><?php echo $address; ?></td></tr>
      <tr><th>Credit Card</th><td><?php echo $ccard; ?></td></tr>
      <tr><th>Date of Birth</th><td><?php echo $dob; ?></td></tr>
      <tr><th>Gender</th><td><?php echo $gender; ?></td></tr>
      <tr><th>Phone</th><td><?php echo $phone; ?></td></tr>
      <tr><th>Track</th><td><?php echo $track; ?></td></tr>
      <tr><th>Statement</th><td><?php echo $statement; ?></td></tr>
    </table>

    <h2>All Form Data</h2>
    <ul>
      <?php
      foreach ($_POST as $key => $value) {
          if (is_array($value)) {
              $value = implode(", ", $value);
          }
          echo "<li><strong>" . safe($key) . ":</strong> " . safe($value) . "</li>";
      }
      ?>
    </ul>

Tip: Save this PHP code in a file named params.php and place it in the same server directory as your HTML file, or adjust the path as needed.

Simple Build & Test Guide

  1. Save this HTML file to your web folder.
  2. Create a separate file named params.php using the template above.
  3. Update the form action to your absolute Codd path.
  4. Upload both files to your server.
  5. Open the HTML page in your browser, fill out the form, and submit it.
  6. Verify that the PHP page displays raw data, individual values, and the full field loop.
Debugging hint
If no values appear, check that each input has a proper name and that the form uses method="post".
Security hint
Never print user input directly without escaping it first.

Critical Thinking Questions

Answer these during development and compile your responses into a Word document.

1. What is the difference between a field’s id and name, and why does PHP depend more directly on name?
2. Why is POST a better fit than GET for this activity?
3. Why is print_r($_POST) useful during debugging, and why should it usually not remain in a finished production page?
4. Why is htmlspecialchars() important when displaying form data back to the user?
5. How does replacing radio buttons with a dropdown change usability, speed, and layout?
6. Which part of your page shows the strongest evidence of thoughtful design rather than just minimum completion?
7. If you rebuilt this activity in Flutter, what widgets and validation flow would you use?
8. What additional field or validation rule would improve this assignment most, and why?
